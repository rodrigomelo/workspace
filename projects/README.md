# Projects

Cada projeto deve seguir esta estrutura:

```
{projeto}/
├── design/         (specs, wireframes, user journeys)
├── code/           (src, tests, scripts, Dockerfiles)
├── qa/             (test plans, resultados, bugs)
├── docs/           (contexto, handoffs, lições aprendidas)
└── CHANGELOG.md   (o que foi feito, por quem, quando)
```

## Começando um novo projeto

1. Copie o template: `cp -r projects/.template projects/{nome-do-projeto}/`
2. Preencha os arquivos com informações do seu projeto
3. Documente handoffs no `HANDOFFs.md`
4. Adicione link aqui na lista de projetos ativos
