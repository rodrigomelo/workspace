## Data Flow

### 1. Data Refresh (Cron Job)
```
Every 30 minutes:
  fetch_data.py → football-data.org API → JSON files
```

### 2. User Request
```
User visits site → /api/* → reads JSON → serves HTML
```

### 3. Calendar Subscription
```
User subscribes to .ics → server reads matches.json → generates calendar
```

## Key Features

| Feature | Description |
|---------|-------------|
| **Zero API Calls** | Frontend never calls external APIs |
| **30-min Cache** | Data refreshes every 30 minutes via cron |
| **Offline Fallback** | Works even if API is down |
| **Calendar Sync** | Auto-updating .ics subscription |
| **Fast Loading** | Serves static JSON, no wait for API |

---

## API Endpoints

| Endpoint | Description | Data Source |
|----------|-------------|-------------|
| `/api/matches` | All matches | `matches_*.json` |
| `/api/standings` | League table | `standings.json` |
| `/api/calendar.ics` | Calendar feed | `matches_scheduled.json` |

## Cron Setup

```bash
# Run every 30 minutes
*/30 * * * * cd /path/to/palmeiras-web/data && python3 fetch_data.py
```

## Benefits

✅ **No Rate Limiting Issues** - API called only 1x per 30 min  
✅ **Fast Performance** - Serves cached JSON instantly  
✅ **Reliable** - Works offline if API is down  
✅ **Clean Architecture** - Clear separation of concerns  
✅ **Easy Debugging** - Check JSON files directly
