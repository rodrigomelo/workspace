# Architecture Decision Records (ADRs)

Registro das decisões arquiteturais e de design tomadas.

## Formato

Cada ADR é um Markdown com:

- **Título**
- **Status**: Proposed, Accepted, Deprecated, Superseded
- **Context**: Por que precisamos decidir?
- **Decision**: O que decidimos e por quê
- **Consequences**: Prós, contras, impacto

## Nomeação

`YYYY-MMDD-nome-da-decisao.md`

## Quando criar

- Ao mudar uma arquitetura existente
- Ao adotar uma nova tecnologia/padrão
- Quando uma decisão impacta múltiplos agentes/projetos

## local

Salvar em `knowledge/decisions/`
